Tokenization

Breaking text into tokens: the first step before turning language into numbers.

Tokenization is the process of splitting raw text into smaller units called tokens, which are then mapped to numerical IDs for machine learning models. The way text is tokenized directly impacts model performance, efficiency, and understanding.

Word-level Character-level Subword-level

Word-level Tokenization

Idea: Split text into individual words using spaces or punctuation.

Sentence:
"I love NLP"

Tokens:
["I", "love", "NLP"]

Advantages

  • Simple and intuitive
  • Easy to interpret and debug

Disadvantages

  • Large vocabulary size
  • Cannot handle unseen (out-of-vocabulary) words
  • Struggles with typos and variations

When it shines

  • Small datasets with controlled vocabulary
  • Basic NLP tasks and rule-based systems

Character-level Tokenization

Idea: Break text into individual characters.

Sentence:
"hi"

Tokens:
["h", "i"]

Advantages

  • No out-of-vocabulary problem
  • Handles typos and rare words naturally

Disadvantages

  • Longer sequences → more computation
  • Loses semantic meaning at higher levels

When it shines

  • Languages with complex morphology
  • Tasks involving noisy or misspelled text

Subword-level Tokenization

Idea: Split words into smaller meaningful units (subwords) to balance vocabulary size and flexibility.

Word:
"unhappiness"

Tokens:
["un", "happi", "ness"]

Common methods

  • BPE (Byte Pair Encoding)
  • WordPiece
  • Unigram Language Model

Advantages

  • Handles unseen words effectively
  • Smaller vocabulary than word-level
  • Captures meaningful patterns

Disadvantages

  • More complex to implement
  • Tokens may not always be human-readable

When it shines

  • Modern NLP and GenAI models (e.g., Transformers)
  • Large-scale multilingual datasets
