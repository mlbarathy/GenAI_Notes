Foremost models

Foundational Transformer-era architectures: bidirectional encoders, seq2seq denoising, and autoregressive GPT-style LMs.

Foremost models here means landmark systems built on the Transformer stack after the original “Attention Is All You Need” paper—covering both encoder-rich (BERT), encoder–decoder (BART), and decoder-only generative pre-training (GPT‑1/2/3).

Overview BERT BART GPT‑1 GPT‑2 GPT‑3

How these models relate

All of the models below use Transformer layers, but they differ in objective (what is predicted during training), directionality (one token vs full context), and stack layout (encoder only, decoder only, or both).

  • BERT — encoder-only; masked language modeling learns bidirectional context.
  • BART — encoder + decoder; text is corrupted then reconstructed (denoising).
  • GPT‑1, GPT‑2, GPT‑3 — decoder-only left-to-right language modeling at increasing scale; emphasis shifts from fine-tuning (GPT‑1) to broad few-shot / in-context use (GPT‑3).

BERT

BERT (Bidirectional Encoder Representations from Transformers) uses a Transformer encoder stack. Tokens attend in both directions within a segment, so each representation can use full left-and-right context.

Pre-training (typical)

  • Masked LM (MLM) — some tokens are hidden; the model predicts them from context.
  • Next sentence prediction (NSP) — in original BERT, whether sentence B truly follows sentence A (later work often drops or replaces NSP).

Downstream use

Fine-tune on task-specific heads for classification, NER, QA, etc. Strong when you need rich token- or sentence-level representations rather than open-ended long generation.

BART

BART combines a bidirectional encoder (BERT-like) with an autoregressive decoder (GPT-like). The encoder reads a corrupted version of the text; the decoder generates the original.

Training idea

  • Denoising autoencoder — corruptions include token masking, deletion, permutation, document rotation, etc.; the model learns to reconstruct clean text.

Good fit

Summarization, translation, and other sequence-to-sequence tasks where output text should be grounded in a full input span.

GPT‑1

GPT (Generative Pre-Training) introduced a decoder-only Transformer trained as a standard left-to-right language model on unlabeled text, then adapted with supervised fine-tuning on labeled tasks.

Recipe

  • Unsupervised pre-training — predict the next token given prior tokens.
  • Supervised fine-tuning — add a light task head (e.g. classification) with labeled data.

Showed that a single pre-trained LM could transfer across tasks with minimal architectural changes.

GPT‑2

GPT‑2 scaled up data and model size and demonstrated strong zero-shot behavior: the same LM, without task-specific fine-tuning, could often follow instructions implied by prompt formatting (e.g. continue a translation-style prefix).

  • Still decoder-only next-token prediction.
  • Showed that scale + diverse web text → broader “emergent” task patterns at inference time.

GPT‑3

GPT‑3 pushed scale further and popularized in-context learning: condition the model with a few examples (or instructions) in the prompt and read off completions without weight updates.

  • Few-shot / one-shot / zero-shot regimes defined by how many labeled examples appear in the prompt.
  • Architecturally still a large autoregressive Transformer decoder.

Practically, GPT‑3 anchored the “prompting + API” era for generative NLP and downstream product builders.
