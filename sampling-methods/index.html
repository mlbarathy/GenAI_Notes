Sampling Techniques

How models choose the next token: logits → softmax → decoding strategies.

Logits Softmax Argmax Temperature Top-k Top-p Comparison Use Cases

🔹 Logits

In models like GPT, when predicting the next word/token:

  • Model processes input text
  • Outputs a vector of logits (one score per possible token)
  • These logits are converted into probabilities using Softmax function
  • A token is selected (based on sampling strategy)
Input → Model → Logits → Softmax → Probabilities → Argmax → Token Selection

How It Works

  • Sort tokens by probability
  • Keep only top K
  • Sample randomly from those

🔹 Softmax

Softmax : In Generative AI, the Softmax function is the step that converts a model’s raw outputs (logits) into probabilities that sum to 1. Without Softmax, the model just has arbitrary scores. With Softmax, it can decide how likely each token is.

  1. Converts scores into probabilities for Sample outputs & Make decisions
  2. Enables randomness & creativity : Sampling (diverse outputs) & Controlled randomness (via temperature)
  3. Preserves ranking : Higher logit → higher probability

Effect of Temperature on Softmax

Temperature modifies logits before Softmax:

  • Low temperature (0.2) → sharper probabilities (more deterministic)
  • High temperature (1.5) → flatter probabilities (more creative/random)

Key takeaway

  • Logits → raw scores
  • Softmax → converts to probabilities
  • Probabilities → used for token selection

🔹 Argmax

Selecting the token with the highest probability (or highest logit).

Input → Model → Logits → Softmax → Probabilities → Argmax → Output token

Example

Let’s say the model predicts the next word after:

"I am feeling"

Probabilities After Softmax:

Token   Probability
happy   0.65
tired   0.25
hungry  0.10

👉 Argmax output: happy (because it has the highest probability)

Key Characteristics

  • Deterministic : Same input → Always same output and No randomness
  • Fast & Simple : Just pick the max value
  • No Creativity : Can be Repetitive & Always chooses the most likely word

Argmax vs Sampling

Method          Behavior
Argmax          Picks highest probability
Sampling        Picks based on probability distribution

Sampling Techniques

Sampling techniques are used to generate text from a model. They are used to control the randomness and creativity of the model's output.

  • Temperature : Controls how random or creative the model’s output is.
  • Top-k : Selects the top k most likely tokens from the model's output.
  • Top-p : Selects the tokens until the cumulative probability is greater than or equal to p.

Temperature

Simple Definition

  • Controls how random or creative the model’s output is.
  • Low Temperature : Deterministic, Predictable
  • High Temperature : Creative, Diverse, Sometimes Chaotic

⚙️ How It Works Internally

Model generates logits → applies at Softmax → gets probabilities.

Logits : Logits are the raw, unnormalized scores produced by a model before converting them into probabilities.

  • T < 1 : Sharp Distribution (Less Randomness)
  • T > 1 : Flatter Distribution (More Randomness)

Real-World Use Cases

  • Chatbots (low T for accuracy)
  • Content generation (high T for creativity)
  • Code generation (very low T)

Example

Prompt: "Write a tagline for a hotel"

T = 0.2 :  “Comfortable stay at affordable prices”
T = 0.9 :  “Where sunsets whisper luxury into every moment”

Pitfalls

  • High T : Hallucinations
  • Low T : Repetitive Output

2. Top-k Sampling

Model selects next token from top K most probable tokens only.

⚙️ How It Works

  • Sort tokens by probability
  • Keep only top K
  • Sample randomly from those

📊 Impact

K value    Behavior 
1          Same as argmax
10         Limited diversity
50         Balanced
100+       High diversity

💡 Example

Token    Prob 
luxury   0.4
comfort  0.3
stay     0.2
cheap    0.1

top_k = 2 :  {luxury, comfort}
top_k = 4 :  all tokens

Pitfalls

  • Small K : Repetitive
  • Large K : Noisy

op-p (Nucleus Sampling)

Select tokens until cumulative probability ≥ p.

How It Works

Instead of fixed K:

  • Keep adding tokens until sum ≥ p

📊 Impact

p Value    Behavior 
0.5        Conservative
0.8        Balanced
0.95       Creative

💡 Example

Token   Prob 
luxury  0.4
comfort 0.3
stay    0.2
cheap   0.1

p = 0.7 → {luxury, comfort}
p = 0.9 → {luxury, comfort, stay}

🔥 Comparison Table

| Method      | Randomness  | Control     | Best For             |
| ----------- | ----------- | ----------- | -------------------- |
| Temperature | Medium–High | Fine-tuning | Creativity control   |
| Top-k       | Medium      | Hard cutoff | Controlled diversity |
| Top-p       | Medium–High | Adaptive    | Production systems   |
| Argmax      | None        | Full        | Deterministic tasks  |

✅ For use cases (data + APIs)

| Use Case              | Recommended         |
| --------------------- | ------------------- |
| SQL generation        | temperature=0       |
| Hotel recommendations | temp=0.5, top_p=0.9 |
| Chatbot               | temp=0.7            |
| Creative marketing    | temp=0.9            |

⚙️ Code Example (Python)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Suggest hotel tagline"}],
    temperature=0.8,
    top_k=50,
    top_p=0.9
)
